Athena Custom Api
Introduction
Use the Athena API to programatically create\update incidents and request.
The Athena Api is a Rest service that allows you to perform "REST" operations to create and modify objects within the Athena system. All
data is returned as JSON and is only transmitted over HTTPS
How do I get access to the api?
In order to access the api you need to have a client Id. To do this just submit a ticket to the help desk requesting access to Athena Api this
will get routed to the proper team which will provide you with a client Id. Once you have a client Id you can use that in combination with
active directory credentials to log in.
Authentication
The Athena Api uses token authentication, in order to get a token you need to provide your username, password, grant_type and client id in
a post request with a content-type of application/x-www-form-urlencoded to the authentication endpoint
Curl command
If the username and password and client Id is correct you will recieve a jwt token, you need to add this token to the authorization header in
all subsequent request you make to Athena Api Endpoints
Type Projections
In Athena Type Projections are definitions that define what relationships are included within an object. For example if you have an incident
ticket object an example of a relationship would be the affected user or the assigned to user. Within an incident form there are many
relationships however if you were to return all related items on every call it would be very taxing on the system. In order to control which
relationships are included we use type projections, which are definitions that determine which relationships to include. These definitions are
created and stored in the Athena system and can be referenced by their Id which is a Guid.
How do I know which type projection to use and how do I get the Id of a type projection?
In most cases you can use projection helpers that we have defined for the main ticket objects within Athena. These are readable names that
will atomatically map to a predefined type projection in Athena. In a rare case that you need access to a related item that is not defined in
our predefined list you can contact the Athena Admin group

Unauthorized vs Forbidden
If you make a call to any resource and you do not have a valid token (includes expired tokens) you will recieve a 401 Unauthorized
response. If you try to access a resource that you are not authorized to access you will recieve a 403 Forbidden response

How to Retrieve Tickets
In Athena there are two types of tickets. Incidents and Service Request. Incidents are break fix request while service request are request from a user for something to be provided.
There are multiple ways to retrieve tickets. If you know the unique identifier (entity id) of a ticket you can use that to retrieve a single ticket, you can also use simple filter criteria to retrieve a ticket

GET
/v1/object/{id}
This endpoint allows you to retrieve any object within Athena by its entity id.

Using $expand
When you retrieve an object and specify a type projection the default response will include all related entities. If you wanted to only include
certain related entities you can specify a list of relationships to include seperated by a comma in the expand parameter such as ?
typeProj=incidentform&$expand=affectedUser,createdByUser
Using $select
The $select option specifies a subset of properties to include in the response body. For example if you only want to include the title and
description of the ticket you would use ?typeProj=incidentform&$select=title,description . If you want to specify the properties on a
related object you would write it like this: ?typeProj=incidentform&$select=affectedUser/displayName

Providing values for enum list items
In the above example you will notice that we specified the name for the location and the id for the area the data type for both of
these properties is an enum data type. for enum data types you can either specify the id or name. Please note that specifying the id
will always be more performant as the application does not have to perform a look up.
